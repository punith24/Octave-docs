    Appium Installation

    Downloading Appium

    • Download appium desktop version from here

    Download the .dmg file for mac.

    Downloading Node.js

    • Download the latest version of node.js from here.

    Downloading Xcode

    • Download and install XCODE from the App Store

    Install Command Line Tools

    • Open the Terminal
    • Enter “xcode-select –install”
    • You will be presented with the below popup. Click on Install button.
    • Wait for the components to be installed and click on Done.

    Add Developer Account

    • Add your developer account to Xcode. You can add your developer account by selecting Preferences… from the Xcode menu.

    • Navigate to the Account tab and click on the plus button at the bottom left.

    • Choose Add Apple ID… from the list of options after clicking on the plus button. Enter your credentials and click Add.

    External Dependencies

    Prerequisite have HomeBrew installed.

    Name Purpose Command
    ideviceinstaller ideviceinstaller is a system requirement for working on ios real devices for iOS 9 brew install ideviceinstaller
    carthage carthage is used to interact with the XCUITest brew install carthage
    ios-deploy ios-deploy is used to deploy application on real device for iOS 10 npm install -g ios-deploy
    device-console device-console is used to get the logs of the device npm install -g device-console
    libimobiledevice A cross-platform protocol library to communicate with iOS devices brew install libimobiledevice --HEAD
    xcpretty Ruby gem to get more readable xcode logs gem install xcpretty

    Appium-Doctor

    • We can install appium-doctor to validate our setup for appium . Once we run appium-doctor it will check that all the required dependencies are installed. 
    npm install -g appium-doctor
    • Once installation is done appium-doctor can be run to check the appium installation.
    appium-doctor

    WebDriverAgent Configuration

    • Navigate to appium location and from there to /node_modules/appium-xcuitest-driver/WebDriverAgent/
    cd usr/local/lib/node_modules/appium/node_modules/appium-xcuitest-driver/WebDriverAgent/
    • Run the below command after navigating to the above location
    mkdir -p Resource/WebDriverAgent.bundle

    If Permission is denied then give the same command using sudo as sudo mkdir -p Resource/WebDriverAgent.bundle This will prompt you for the password first and then proceed with the command

    • Run the below command as the next step to webdriver configuration
    sudo sh ./Scripts/bootstrap.sh -d

    • Type open . in the terminal to open the current location

    • Navigate Back to the following location and add permissions 

    cd usr/local/lib/node_modules/appium/node_modules/ (Press Enter)
sudo chmod -R 777 . appium-xcuitest-driver/*

    • Open the WebDriverAgent.xcodeproj with xcode (Right click on it to open with xcode).

    • Once opened build the WebDriverAgent.xcodeproj ,for both the WebDriverAgentLib and WebDriverAgentRunner targets, select "Automatically manage signing" in the "General" tab, and then select your Development Team. This should also auto select Signing Ceritificate. The outcome should look as shown below:

    • Xcode may fail to create a provisioning profile for the WebDriverAgentRunner target:

    • Navigate to Build Settings Tab and change the "Product Bundle Identifier" from com.facebook.WebDriverAgentRunner

    • Once done with the above step the provisioning profile error will go.

    • Finally, you can verify that everything works. Build the project:

    xcodebuild -project WebDriverAgent.xcodeproj -scheme WebDriverAgentRunner -destination 'id=<udid>' test
    • If this was successful, the output should end with something like:
