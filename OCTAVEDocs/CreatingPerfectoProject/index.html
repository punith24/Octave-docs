    Creating Mobile Test Scripts from Eclipse IDE

    Creating Project On Eclipse

    • Create a new Java Project and then a new package with first name starting from "adapters". For e.g. adapters.mobile
    • We need adapter class, this class contains all the desired capabilities such as deviceName, Perfecto Mobile Cloud Credentials. So this class can be directly used.
    • Next step is to create a package where Test Scripts are written.
    • In order to maintain modularity, we write the Common utils like Screenshot in a different package, and invoke them where ever required.
    • A sample Project with the correct modular structure is uploaded in the References

    Additional Features from Perfecto

    We can use some of the additional features Provided by Perfecto(in PerfectoLabUtils and WindTunnels class file) to enhance the real user experience in testing, some of them are:

    • Network Virtualization: This feature enables the testers to emulate real world network conditions as part of the test script. We can set the network profile to 2g, 3g or 4g

      The code snippet for this is written in PerfectoLabUtils.java class:

      PerfectoLabUtils.networkvirtualizationstart(driver, "2g_gprs_good");

      Some of the other profiles are listed in the Perfecto Mobile site. Click this link to see more network profiles.

    • Start Wind: This features is used to collect the device vitals such as CPU usage, bandwidth in/out, memory cache,etc. during a period of test execution or complete execution The code snippet for this is written in PerfectoLabUtils.java class:

      PerfectoLabUtils.startWind(driver);

      Similarly it should be stopped, so that the vitals is captured from using the below code:

      PerfectoLabUtils.stopWind(driver);
